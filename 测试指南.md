## 🧪 本地测试方法

### 快速测试（推荐）

1. **打开 PowerShell 或 CMD**
2. **进入项目目录**：
   ```powershell
   cd F:\github\Personal-Homepage
   ```
3. **启动服务器**：
   ```powershell
   python -m http.server 8000
   ```
4. **访问网站**：
   - 打开浏览器访问：`http://localhost:8000`
   - 或直接访问：`http://localhost:8000/index.html`

### 测试步骤

1. **测试主页**：
   - 访问 `http://localhost:8000/index.html`
   - 检查知识库和项目集列表是否显示

2. **测试知识库**：
   - 点击顶部栏"知识库"或访问 `http://localhost:8000/knowledge.html`
   - 检查左侧导航栏是否显示
   - 检查中间是否显示文章列表
   - 点击文章，检查内容是否正常显示
   - 检查右侧目录是否正常显示

3. **测试项目集**：
   - 点击顶部栏"项目集"或访问 `http://localhost:8000/projects.html`
   - 检查功能是否正常

4. **测试主题切换**：
   - 点击顶部栏的太阳按钮
   - 检查暗色模式是否正常切换

5. **测试浏览器导航**：
   - 点击文章后，使用浏览器后退按钮
   - 检查是否能正常返回列表

## ⚠️ 常见问题

### 问题：页面空白
- **原因**：直接用 `file://` 打开 HTML 文件
- **解决**：必须使用 HTTP 服务器（如 `http://localhost:8000`）

### 问题：文章列表为空
- **检查**：打开浏览器控制台（F12），查看错误信息
- **检查**：`data/articles-manifest.json` 文件是否存在
- **检查**：文章文件路径是否正确

### 问题：中文路径404
- **检查**：确保 `.nojekyll` 文件存在
- **解决**：重新部署到 GitHub Pages
